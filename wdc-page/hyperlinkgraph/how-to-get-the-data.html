Index/Arc Files of the Page Graph

The page graph, which is an unaggregated representation of the hyperlink graph of the used Web crawl is currently hosted using Amazon Simple Storage Service (S3). Due to the large size of the graph, we currently cannot make it completely public, but we offer the graph within a Requester Pays Bucket. In a nutshell you need an AWS account to download the files using s3cmd. Please note: The costs for downloading will be charged by Amazon to your AWS account. Based on the S3 pricing the estimated download costs for index and arc files (45 + 331GB) are around 45 USD. When operating on the data within the same region (US Oregon) is free of charge.
S3cmd is available as standard package for almost all UNIX systems (e.g. Ubuntu apt-get install s3cmd). You can either request all files at once using the command s3cmd get --recursive s3://wgc-2012-data/indexall/ for the index files and s3cmd get --recursive s3://wgc-2012-data/network/distinctOrderedNetwork/ for the arc files. Or request the files separately using the following two list:

  • Arc file list

WebGraph Format

We have used the WebGraph Framework of Sebastiano Vigna for most of the calculations. The required format of the page graph is also available as download. It consists of 3 files. .graph, .offsets, .properties. The files (together 57GB) can be downloaded from our S3 Requester Pays Bucket using s3cmd get --recursive s3://wgc-2012-data/webgraph/original/. You can also request each file individually using:

  • s3cmd get s3://wgc-2012-data/webgraph/original/network.graph (52GB)
  • s3cmd get s3://wgc-2012-data/webgraph/original/network.offsets (4GB)
  • s3cmd get s3://wgc-2012-data/webgraph/original/network.properties (1.5MB)
Using the WebGraph Framework, which can be grabbed from Maven Central these files can be loaded using the following line of code: BVGraph graph = BVGraph.loadMapped(baseName, new ProgressLogger()).

Pajek NET Format

Most calculations and experiments we did on the PLD Graph where done using Pajek. Pajek supports the Pajek NET Format. The Pajek PLD Graph (2.3GB) can directly imported into Pajek after unzipping. To process the graph in a feasable time we recommend to grant Pajek around 32GB of RAM.

Popularity Ranks

Beside the pure graph files, we also calculated for each host in the host graph the harmonic centrality, indegree centrality, Katz's index, and Page Rank. The ranking can be browsed here, or the data can be downloaded:

  • Harmonic Centrality: http://zeist.informatik.uni-mannheim.de/wdc-graph/hostgraph-h.tsv.gz
  • Indegree: http://zeist.informatik.uni-mannheim.de/wdc-graph/hostgraph-indegree.tsv.gz
  • Katz: http://zeist.informatik.uni-mannheim.de/wdc-graph/hostgraph-katz.tsv.gz
  • PageRank: http://zeist.informatik.uni-mannheim.de/wdc-graph/hostgraph-pr.tsv.gz
