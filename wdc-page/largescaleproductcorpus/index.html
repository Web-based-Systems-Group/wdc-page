Web Data Commons - Gold Standard for Large Scale Product Matching

Generating a Gold Standard for Large Scale Product Matching
Anna Primpeli
Christian Bizer

We present the WDC Gold Standard for Large Scale Product Matching, a gold standard containing more than 3.1 million entities describing product offers. For the gold standard creation we make use of the supervision found on the web in the form of schema.org offer entities annotated with schema.org identifier information. In the following sections, the necessary cleansing steps for creating the large scale product matching gold standard are explained as well as possible use cases and experimental baseline results.

1. About the WDC Gold Standard for Large Scale Product Matching

Both N:1 and N:M matching scenarios using web data describing product offers are hard intergration tasks mostly because of the sparse and noisy nature of web data. Although there exist product data sets which can be used to evaluate product matching methods (e.g. the WDC Product Corpus and the Abt-Buy data set), they contain data that either refer to a small group of unique products or derive from a small number of sources. The availability of large scale product information has been so far restricted to large marketplaces such as ebay and amazon. However, the application of supervised representation learning methods like supervised neural networks require a large annotated corpus. Therefore the need for a gold standard for large scale product matching becomes apparent.

Popular annotation vocabularies, like schema.org, propose a set of properties to annotate identifiers of offer entities. In addition, google encourages the annotation of product identifiers with properties such as schema.org/Product/sku, gtin8, gtin13, gtin14, and mpn for more enhanced search results. The application of these annotation practices can be considered as inexpensive supervision offered in the web. We profile the adoption of identifier-related schema.org product properties using the Web Data Commons November 2017 schema.org/Product subcorpus, perform a series of cleansing and clustering steps, and create the WDC Gold Standard for Large Scale Product Matching containing 3.1 million entity offers clustered in 172,581 id-groups.

2.Initial Corpus

We use the November 2017 WDC schema.org/Product subcorpus and extract the offer entities that contain schema.org identifier properties. We consider the following schema.org/Product and schema.org/Offer properties as identifier related properties: sku, mpn, identifier, productID, gtin14, gtin13, gtin12, and gtin8.

3.Gold Standard Creation

Considering the noisy nature of web data, we perform a series of cleansing steps to ensure the quality of our gold standard. During the cleansing process we identify several false practices while annotating offer entities and develop different cleansing steps to overcome them. Below we provide details about the different false annotation practices as well as the corresponding cleansing steps and the effect they had on the resulting corpus in terms of size.

Misuse of schema.org vocabulary

A large number of offer entities are assigned wrong schema.org vocabulary terms such as schema:IndividualProduct/sku and schema:Product/Offer/sku. Despite the wrong vocabulary usage, such terms still reveal identifying information. We capture this by applying a number of regex patterns on the properties of the offer entities.

Value Normalization

The next filtering step is to remove offer entities which have been annotated with invalid identifier values. As we do not possess an extensive list of all valid product identifiers, we consider the definitions of the relevant schema.org identifier properties and apply value normalization. First, all non-alphanumeric characters and initial 0s of the identifier values are removed. Considering the suggested length and structure of the identifying properties, we remove the values that after normalization are shorter than 8 characters or longer than 25 characters. Finally, we remove the values that consist only out of alphabetic characters.

Usage of multiple valid identifiers to indicate the same product

Across web sites different valid identifier values might be used to indicate the same product, e.g. one web site might add a gtin13 annotation while another one might add the mpn number to an schema.org offer entity. For capturing the different identifying values of an offer entity, we depict our data as a network graph with the distinct identifier values as vertices connected by "sameAs" edges in the case they appear in the same schema.org offer entity. The connected components of the graph form the ID-clusters of our gold standard.

Usage of product taxonomy identifiers as unique product identifiers

Another common error is to annotate product taxonomy identifiers, such UNSPSC, as product identifiers. To consistently detect and remove those errors we make use of the network graph structure of our data. The taxonomy identifiers can be recognized as the nodes with a high degree and a very small clustering coefficient. The high degree reveals that the taxonomy identifier co-occurs with many other identifier values while the small clustering coefficient shows that the co-occuring identifier values do not appear in the same offer entities and thus describe different products.

4.Gold Standard Profiling

After performing a series of cleansing steps described in the previous section we end up with 267,530 distinct identifier values clustered into 172,581 ID-clusters with 3.1 million offers in total. In this section we present different statistics concerning the structure of the created ID-clusters w.r.t. identifier values count, offer entities count and provenance.

5.Gold Standard Data

We offer the WDC gold standard for large-scale product matching of the 3.1M annotated offers as json objects including the following information for every offer:

  • url: The URL where the annotated offer entity was found (considering the November 2017 Common Crawl Corpus)
  • nodeID: The subject assigned by the WDC Extractor. The combination of url and nodeID can be used as unique keys for every annotated entity.
  • clusterID: The ID of the cluster where the offer belongs.
  • identifiers: A list of all annotated identifier values of the offer entity together with their corresponding properties.
  • schema.org_description: A list of all annotated schema.org product descriptive properties such as name, description, brand etc. and their values.

In addition we offer a English subcorpus of the gold standard by filtering the following public suffixes: com, net, co.uk, us, and org. The English subcorpus of our gold standard contains 1,726,910 annotated offer entities.

We provide metadata for the created ID-clusters in the form of json objects containing the following information:

  • id: The cluster identifier which can be used as key.
  • size: The size of the ID-cluster in terms of entity offers count.
  • id_values: The distinct identifier values of the clustered offers of the ID-cluster.

Finally we provide the html content of the crawled URLs containing the clustered annotated offer entities in the form of json objects with the following properties.

  • url: The URL whose content was fetched by the Common Crawl crawler (November 2017).
  • fileName: The name of the WARC file of Common Crawl that contains the related HTML response.
  • offset: The index of the HTML response in the related WARC file.
  • content: The HTML response to the call given the related URL.
